# 🚨 ReportSystem — 檢舉系統

使用者可透過右鍵選單檢舉違規訊息，系統會透過 AI 自動判斷訊息嚴重程度（0-5 級），並依據伺服器規則建議處分動作。內建冷卻時間與黑名單機制防止濫用。

## 使用方式

> **如何檢舉：**在訊息上 **右鍵**（手機長按）→ **應用程式** → **檢舉訊息**，填寫檢舉原因後送出。系統會自動通知管理員並觸發 AI 審核。

## 管理員設定指令（/report）管理員

| 指令 | 說明 |
| --- | --- |
| `/report settings` | 查看或修改檢舉系統的各項設定 |
| `/report blacklist-role` | 管理檢舉黑名單身分組（新增 / 移除 / 查看） |
| `/report set-server-rules` | 設定 AI 審核所依據的伺服器規則 |

## 可設定項目

| 設定項目 | 說明 | 預設值 |
| --- | --- | --- |
| `檢舉通知頻道` | 檢舉紀錄與 AI 判斷結果的接收頻道 | 未設定 |
| `處分通知頻道` | 執行處分後公告的頻道 | 未設定 |
| `檢舉回覆訊息` | 提交檢舉後顯示給檢舉人的確認訊息 | 感謝您的檢舉... |
| `檢舉頻率限制` | 同一使用者兩次檢舉之間的冷卻時間（秒） | 300 秒 |
| `檢舉通知訊息` | 通知頻道中額外 @提及的角色或內容 | @Admin |

## AI 審核流程

當收到檢舉後，系統會自動執行以下流程：

> 1. 擷取被檢舉訊息的內容與附件
> 2. 蒐集該用戶最近 10 則歷史訊息作為上下文
> 3. 根據伺服器規則，由 AI 判斷違規等級（0-5）
> 4. AI 給出建議處分（禁言 / 踢出 / 封鎖）與理由
> 5. 管理員可直接點按鈕執行 AI 建議，或手動選擇處分

## 管理員操作按鈕

檢舉通知送達檢舉頻道後，管理員可使用以下操作按鈕：

| 按鈕 | 說明 |
| --- | --- |
| `執行 AI 建議處置` | 一鍵執行 AI 給出的建議處分 |
| `封鎖` | 手動封鎖被檢舉的用戶（可設定刪除訊息時間與封鎖時長） |
| `踢出` | 手動踢出被檢舉的用戶 |
| `禁言` | 手動禁言被檢舉的用戶（可設定時長） |
| `查看前10則訊息` | 查看被檢舉者在該頻道的歷史訊息 |
| `拔除檢舉人檢舉權限` | 惡意檢舉時可撤銷該用戶的檢舉資格 |
| `拒絕檢舉` | 拒絕本次檢舉，並發送私訊通知檢舉人 |

## 黑名單機制

> **檢舉黑名單：**管理員可透過 `/report blacklist-role` 設定「檢舉黑名單身分組」。擁有該身分組的成員將無法提交檢舉。適用於防止特定用戶濫用檢舉功能。
