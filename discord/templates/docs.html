🚀 快速入門

歡迎使用本機器人！只需幾個簡單步驟即可開始：
  1. 點擊首頁的 「邀請機器人」 按鈕將機器人加入你的伺服器
  2. 機器人加入後，使用 /help 查看所有可用指令
  3. 使用 /tutorial 查看互動式教學指南
  4. 前往 伺服器面板 進行進階設定
提示：本機器人同時支援斜線指令 (/command) 和傳統前綴指令 (y!command)。

⌨️ 指令前綴

預設前綴為 y!。每個伺服器可以透過 y!setprefix 設定自定義前綴。你也可以透過 @提及機器人 來查看目前的前綴設定。

🛡️ Moderate — 管理工具 管理員

核心管理模組，提供完整的使用者管理工具。支援時間字串解析（如 1h、30m、7d），並可一次對多位使用者執行操作。所有懲處動作會自動整合通知系統。
指令說明
/moderate開啟管理選單，可選擇踢出、封禁、禁言、黑名單等動作
y!moderate y!m前綴版管理指令
y!moderate_reply y!mr對被回覆訊息進行管理
/action-builder建立自訂的管理動作組合
/send-moderation-message手動發送懲處公告
時間格式：支援 s / 秒、m / 分鐘、h / 小時、d / 天、w / 週、M / 月、y / 年，可組合使用如 1d12h。

動作指令字串 (Action String)

y!moderate 和 /multi-moderate 的 action 參數使用動作指令字串格式，可用逗號 , 分隔多個動作（最多 5 個）。也可使用 /action-builder 透過互動介面產生指令字串。
動作格式說明
banban <時長> <刪除訊息時長> <原因>封禁用戶。時長為 0 表示永久，刪除訊息時長 0 表示不刪除
kickkick <原因>踢出用戶
mutemute <時長> <原因>禁言用戶，預設 10 分鐘。也可使用 timeout
unbanunban <原因>解封用戶
unmuteunmute <原因>解除禁言。也可使用 untimeout
deletedelete <警告訊息>刪除訊息，可附帶公開警告（{user} 代表用戶）
warnwarn <警告訊息>在頻道中發送公開警告
send_mod_messagesend_mod_message發送懲處公告到設定的公告頻道。也可使用 smm
force_verifyforce_verify <時長>強制用戶進行網頁驗證（需啟用 ServerWebVerify）
範例：
ban 7d 1d 違規發言 — 封禁 7 天，刪除最近 1 天的訊息
mute 30m 注意行為, warn {user} 請注意你的發言 — 禁言 30 分鐘並公開警告
delete_dm 你的訊息已被刪除, mute 1h 違規內容 — 刪除訊息 + 私訊通知 + 禁言 1 小時
ban 0 0 嚴重違規, smm — 永久封禁並發送懲處公告

🚨 ReportSystem — 檢舉系統

使用者可透過右鍵選單檢舉違規訊息，系統會透過 AI 自動判斷訊息嚴重程度（0-5 級），並依據伺服器規則建議處分動作。內建冷卻時間與黑名單機制防止濫用。

使用方式

如何檢舉：在訊息上 右鍵（手機長按）→ 應用程式 → 檢舉訊息，填寫檢舉原因後送出。系統會自動通知管理員並觸發 AI 審核。

管理員設定指令（/report）管理員

指令說明
/report settings查看或修改檢舉系統的各項設定
/report blacklist-role管理檢舉黑名單身分組（新增 / 移除 / 查看）
/report set-server-rules設定 AI 審核所依據的伺服器規則

可設定項目

設定項目說明預設值
檢舉通知頻道檢舉紀錄與 AI 判斷結果的接收頻道未設定
處分通知頻道執行處分後公告的頻道未設定
檢舉回覆訊息提交檢舉後顯示給檢舉人的確認訊息感謝您的檢舉...
檢舉頻率限制同一使用者兩次檢舉之間的冷卻時間（秒）300 秒
檢舉通知訊息通知頻道中額外 @提及的角色或內容@Admin

AI 審核流程

當收到檢舉後，系統會自動執行以下流程：
1. 擷取被檢舉訊息的內容與附件
2. 蒐集該用戶最近 10 則歷史訊息作為上下文
3. 根據伺服器規則，由 AI 判斷違規等級（0-5）
4. AI 給出建議處分（禁言 / 踢出 / 封鎖）與理由
5. 管理員可直接點按鈕執行 AI 建議，或手動選擇處分

管理員操作按鈕

檢舉通知送達檢舉頻道後，管理員可使用以下操作按鈕：
按鈕說明
執行 AI 建議處置一鍵執行 AI 給出的建議處分
封鎖手動封鎖被檢舉的用戶（可設定刪除訊息時間與封鎖時長）
踢出手動踢出被檢舉的用戶
禁言手動禁言被檢舉的用戶（可設定時長）
查看前10則訊息查看被檢舉者在該頻道的歷史訊息
拔除檢舉人檢舉權限惡意檢舉時可撤銷該用戶的檢舉資格
拒絕檢舉拒絕本次檢舉，並發送私訊通知檢舉人

黑名單機制

檢舉黑名單：管理員可透過 /report blacklist-role 設定「檢舉黑名單身分組」。擁有該身分組的成員將無法提交檢舉。適用於防止特定用戶濫用檢舉功能。

丟棄與撿起

丟棄選項：/item drop 支援以下參數：
• can_pickup — 其他人是否可以撿起（預設是）
• pickup_duration — 可撿起的時間，1~86400 秒（預設 60 秒）
• pickup_only_once — 每位用戶是否只能撿一次（預設否）
• scope — 物品來源範圍（伺服器 / 全域）

觸發模式

新增自動回覆時，可選擇以下幾種觸發模式：
模式說明範例
包含 (contains)訊息中包含觸發字串即觸發觸發 你好 → 「你好嗎」「說你好」皆觸發
完全匹配 (equals)訊息必須與觸發字串完全相同觸發 你好 → 只有「你好」觸發
開始於 (starts_with)訊息以觸發字串開頭觸發 早安 → 「早安大家」觸發
結束於 (ends_with)訊息以觸發字串結尾觸發 晚安 → 「大家晚安」觸發
正規表達式 (regex)使用 Python regex 進行匹配觸發 你好+ → 「你好好好」觸發
提示：觸發字串與回覆內容都支援使用逗號 , 分隔多個值。多個觸發字串為「任一即觸發」，多個回覆則「隨機選擇一個」。

變數說明

隨機與進階語法

語法說明範例
{random}隨機產生 1~100 的整數「你的幸運數字是 {random}」
{randint:min-max}隨機產生 min~max 範圍的整數{randint:1-999} → 隨機 1~999
{random_user}從最近 50 則訊息中隨機選一位非機器人使用者「隨機點名：{random_user}」
{react:emoji}對觸發訊息添加表情反應（不會產生回覆文字）{react:👍} 或 {react:↖️}
{sticker:ID}傳送指定 ID 的伺服器貼圖{sticker:123456789012345678}
取得貼圖 ID：可使用 y!sticker 指令取得伺服器貼圖的 ID。

🚌 twbus — 台灣公車 / YouBike

即時查詢台灣公車到站資訊、路線總覽、YouBike 站點狀態，並支援收藏站牌。公車資料庫每小時自動更新一次。
指令說明
/bus getroute查詢指定路線的所有站牌與到站資訊
/bus getstop查詢指定路線下特定站牌的即時到站資訊
/bus youbike查詢指定 YouBike 站點的可借車數與空位
/bus favorites查看並快速存取你收藏的站牌與 YouBike 站點

互動按鈕

查詢結果的嵌入訊息下方附有操作按鈕：
按鈕說明
🔄重新整理到站資訊（10 秒冷卻）
❤️加入 / 移除最愛（收藏上限可由伺服器設定）
🗺️在 Google 地圖上開啟站牌位置（若有座標）

到站資訊說明

欄位說明
預估到站距離到站的分秒數，若顯示「進站中」表示車輛已進站
預計到站API 給出的預計時間（時:分格式）
車輛狀態列出接近中的車牌號碼及是否已滿
站序此站牌在路線中的順序
座標點擊可開啟 Google Maps

YouBike 資訊說明

欄位說明
可借車數目前可借走的腳踏車數量
可停空位目前可停放的空格數
總停車格站點的總容量
狀態該站是否正常營運
最後更新YouBike 系統回報的資料更新時間
速率限制：每位用戶每 10 秒只能查詢一次，重新整理按鈕同樣受此限制。
最愛上限：預設每人最多 2 個最愛站牌與 2 個最愛 YouBike 站點，上限可由設定檔調整。
