OXWU API Docs

這份文件整理了 OXWU Proxy API 的 REST 端點、WebSocket 事件、範例 JSON，以及常用欄位說明。你可以直接先看範例格式，再往下對照每個鍵值的用途。

OpenAPI JSON: {{ openapi_url }}

Python client 文件: /api/docs/uooxwu

如果你是第一次接這個 API，通常先看 /api/data/warning、/api/data/report、/api/town-map 就夠用了。

Authentication

需要 API Key 的端點，請在 request header 帶入 X-API-Key。和帳號管理有關的頁面則使用 Discord OAuth2 建立的 session cookie。

  • WebSocket: {{ "%.4f"|format(costs.ws_per_sec) }} 點 / 秒，每 5 秒結算一次

cURL

curl -H "X-API-Key: YOUR_API_KEY" \
{{ request.host_url.rstrip('/') }}/api/data/warning

REST Endpoints

GET/api/data/<report|warning>

取得目前快取中的地震報告或地震速報 JSON。

GET/api/screenshot/<report|warning>

取得目前畫面的 PNG 截圖。

GET/api/town-map

取得 town_id 對照表與座標，可搭配 arrival_times / estimated_intensities 使用。

GET/api/me

取得目前登入帳號資訊、剩餘點數與 API Key。

POST/api/me/reset-key

重設目前登入帳號的 API Key。舊的 Key 會立刻失效。

GET/openapi.json

取得 OpenAPI 3.1 規格。

Warning Example

範例 JSON

{
  "ok": true,
  "time": "2026-03-26 09:21:14",
  "location": {
    "latitude": "23.87",
    "longitude": "121.51",
    "text": "北緯 23.87 / 東經 121.51"
  },
  "depth": "20",
  "magnitude": "4.6",
  "maxIntensity": "3級",
  "intensity": "1",
  "eta": "0",
  "list": [
    {
      "id": "1",
      "time": "09:21:14",
      "latlon": "23.87/121.51",
      "depth": "20",
      "mag": "4.6"
    }
  ],
  "url": "file:///.../index.htm",
  "arrival_times": {
    "1001501": 7,
    "1001502": 9
  },
  "estimated_intensities": {
    "1001501": "3級",
    "1001502": "2級"
  },
  "arrival_count": 142,
  "arrival_generated_at": "2026-03-26T09:21:15+08:00"
}

鍵值說明

鍵 型別 說明
okboolean資料是否有效。
timestring這次速報的發生時間，格式為 YYYY-MM-DD HH:mm:ss。
locationobject震央座標與文字描述。
depthstring深度，單位公里。
magnitudestring規模。
maxIntensitystring這次速報的最大預估震度。
intensitystringOXWU 所在地的預估震度。
etastringOXWU 所在地的預估抵達秒數。
listarray同一輪速報內的警報列表歷程，通常後續更新時會持續增加。
arrival_timesobject{ town_id: eta_seconds }，只保留尚未抵達的地區。
estimated_intensitiesobject{ town_id: intensity_label }，各地預估震度。
arrival_countnumber目前 arrival_times 中的地區數。
arrival_generated_atstring這批 ETA / 預估震度的生成時間，ISO 8601。

WebSocket Events

Socket.IO 連線時可使用 X-API-Key header 或 query string 的 api_key。目前會推送以下事件：

  • proxy_report_update
  • proxy_warning_update
  • proxy_warning_updated

事件說明

事件名稱 用途 說明
proxy_report_update 地震報告更新 當 upstream 偵測到新的地震報告時推送，事件內的 data 會包含完整 report payload。
proxy_warning_update 新一輪地震速報 代表新的速報開始，通常可拿來建立新卡片、新訊息，或重設前端目前追蹤中的 warning 狀態。
proxy_warning_updated 同一輪速報後續更新 代表同一輪 warning 的內容有變化，例如警報列表新增、規模修正、位置修正；適合用來更新既有畫面，而不是當成全新速報。

範例事件

{
  "time": "2026-03-26 09:21:14",
  "parts": {
    "year": "2026",
    "month": "03",
    "date": "26",
    "hour": "09",
    "minute": "21",
    "second": "14"
  },
  "url": "file:///.../index.htm",
  "arrival_times": {
    "1001501": 7
  },
  "estimated_intensities": {
    "1001501": "3級"
  },
  "arrival_count": 142,
  "arrival_generated_at": "2026-03-26T09:21:15+08:00",
  "data": {
    "...": "完整 warning payload"
  }
}

鍵值說明

鍵 型別 說明
timestring這次事件對應的時間。
partsobject時間拆分欄位，方便前端少做字串處理。
arrival_timesobject各地預估抵達秒數。
estimated_intensitiesobject各地預估震度。
dataobject完整 warning / report payload。

前端範例

const socket = io("{{ request.host_url.rstrip('/') }}", {
  extraHeaders: {
    "X-API-Key": "YOUR_API_KEY"
  }
});

socket.on("proxy_warning_update", (payload) => {
  console.log(payload.arrival_times);
  console.log(payload.estimated_intensities);
});
